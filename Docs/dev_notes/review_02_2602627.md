# Adversarial Code Review 02

Date: 2026-06-27

Scope: repo review, ignoring the prior review note. Focus areas were coding errors,
assumptions, architecture risks, wheel-reinventions, performance, and quality risks.

## Findings

1. **High: Fusion can replace OCR text with unrelated VLM hallucination.**

   In `src/fusion_ocr/stages/fusion.py`, `_nw_align()` always accepts diagonal
   matches, and `_fuse_lines()` assigns the mapped VLM line with no minimum similarity
   gate. I verified a one-line OCR segment `"Invoice total 10.00"` becomes
   `"Completely unrelated hallucinated sentence"` with `source="fused"`. Geometry is
   gated, but text truth is not.

2. **High: Job/artifact identity can desynchronize from file content.**

   The watcher hashes a mutable input path, then `process()` re-hashes the same path
   later. If a file is still being copied, or an API upload overwrites the same
   filename, job status can be recorded under digest A while artifacts are written
   under digest B. The API upload path has the same shape. Process immutable
   content-addressed copies, or pass and verify the expected digest.

3. **High: The `api` extra is missing the multipart upload dependency.**

   `pyproject.toml` installs `fastapi` and `uvicorn`, but not `python-multipart`,
   while `src/fusion_ocr/api.py` declares a multipart `UploadFile` endpoint. FastAPI
   documents `python-multipart` as required for file/form uploads. The current API
   tests skip if FastAPI is absent and do not catch this.

4. **Medium: Apple Vision failure silently drops geometry instead of falling back.**

   When routing selects Apple Vision, `_run_vision()` catches every exception and
   returns an empty result. That leaves the page with no deterministic boxes, so VLM
   markdown may exist but overlay/search provenance disappears. If Vision is
   preferred, per-page runtime failure should fall back to PaddleOCR or fail loudly.

5. **Medium: Resume fingerprint omits `overlay_font`, so searchable output can stay stale.**

   The recipe includes `granularity`, but not `overlay_font`, even though render uses
   it to build the overlay PDF. Changing from Helvetica fallback to a Unicode font
   will not invalidate the cached render.

6. **Medium: SQLite idempotency is race-prone.**

   `JobStore.upsert_queued()` does `SELECT` then `INSERT`. Two concurrent submits of
   the same content can race into an `IntegrityError` or duplicate processing. Use
   `INSERT OR IGNORE` / `ON CONFLICT` inside one statement.

7. **Medium: Airgap fail-loud behavior is inconsistent.**

   VLM reading correctly re-raises `AirgapError`, but the language probe catches all
   exceptions. A sealed deployment pointed at a remote probe endpoint can silently
   default routing instead of surfacing misconfiguration.

8. **Quality risk: rotated tables are intentionally skipped.**

   The table structure and focused table-read stages exclude rotated pages. That is
   okay as a scoped limitation, but it conflicts with "done" table claims if rotated
   scans are expected in the corpus.
